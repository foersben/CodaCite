# User Interface

The Enterprise Omni-Copilot features a state-of-the-art "Notebook" interface, designed to provide a premium, cohesive experience for document intelligence. This interface bridges the gap between raw data ingestion and sophisticated contextual reasoning, offering users a centralized environment for interacting with their private document graphs.

## Design Philosophy

The interface is built with a focus on **visual excellence** and **functional density**. It utilizes a modern dark-mode aesthetic with glassmorphism elements, ensuring that the workspace feels premium and immersive.

To prioritize readability and accessibility, the interface implements a **1.5x scaling system**. This high-impact design utilizes generous padding, clear typography (Inter), and fluid layouts that adapt seamlessly to varying screen sizes. The result is a professional environment that minimizes cognitive load while maximizing the impact of the extracted insights.

## Core Components

### Sidebar Document Manager

The left sidebar serves as the control center for your knowledge base.

- **Document List**: Provides a real-time view of all ingested documents, allowing users to track the scale and scope of their private graph.
- **Drag-and-Drop Ingestion**: A dedicated "Drop Zone" enables seamless file uploading. By dragging files directly into the browser, users trigger the full extraction pipeline—from coreference resolution to graph anchoring—without leaving the interface.

### Contextual Chat Engine

The central workspace is dominated by a multi-turn conversational interface.

- **Grounded Reasoning**: Every response generated by the system is grounded in the semantic graph extracted from your documents, ensuring high fidelity and reducing hallucinations.
- **Markdown Rendering**: The chat engine utilizes `marked.js` to render complex markdown responses. This includes support for hierarchical headers, formatted code blocks, detailed lists, and mathematical notation.
- **"Copy MD" Integration**: To facilitate external workflows, each assistant response includes a "Copy MD" feature. This intelligently strips metadata and formatting artifacts, allowing users to copy the original, clean markdown to their clipboard for use in reports or documentation.

## Technical Implementation

The frontend is implemented using a lightweight, dependency-free approach with Vanilla JavaScript and CSS, served via Jinja2 templates from the FastAPI backend.

### Responsive Scaling Logic

The interface leverages CSS `clamp()` and relative `rem` units to achieve its fluid 1.5x scaling. This ensures that font sizes, spacing, and component dimensions remain balanced across mobile, tablet, and desktop viewports, maintaining a consistent professional aesthetic.

### Robust Response Parsing

To ensure stability when interacting with the GraphRAG backend, the frontend implements sophisticated parsing logic. It can handle both standard JSON and structured Python-style stringified objects, robustly extracting the `text` content while filtering out signatures and raw metadata. This ensures that the user only sees the finalized, intelligent output of the system.
